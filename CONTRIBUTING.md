# Contributing Guide

Thanks for your interest in contributing.

## 1) Before You Start

- Open an issue to discuss major changes before implementation.
- Keep pull requests focused and limited in scope.
- Preserve the existing directory layout and naming conventions.

## 2) Development Workflow

1. Fork the repository.
2. Create a feature branch:
   - `feature/<short-description>`
3. Implement changes with clear, minimal commits.
4. Validate your changes by running relevant Spark jobs.
5. Open a pull request with context, motivation, and evidence.

## 3) Code Standards

- Follow the project’s current Python style.
- Use clear variable names and avoid unnecessary complexity.
- Keep scripts reproducible and avoid hard-coded machine-specific paths when possible.

## 4) Pull Request Checklist

- [ ] Change is scoped to one concern.
- [ ] README/docs are updated if behavior changed.
- [ ] Commands used for verification are included in PR description.
- [ ] Output screenshots or logs are attached when relevant.

## 5) Reporting Issues

Please include:

- Environment (OS, Python version, Spark version)
- Steps to reproduce
- Expected behavior
- Actual behavior
- Error logs/screenshots

Thanks for helping improve the project.
